 INSTRUCCIONES PARA RECREAR LA BASE DE DATOS (CANACO)
===================================================

1. Crear la Base de Datos
   Nombre: tickets_canaco

2. Crear Tablas Principales (Ejecutar en orden)

-- A) Tabla de Usuarios (Ajustada para Soporte Híbrido)
CREATE TABLE usuarios (
    id SERIAL PRIMARY KEY,
    nombre VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL, -- Soporta texto plano (legacy) y bcrypt
    rol VARCHAR(50) DEFAULT 'tecnico' -- Unificado con el controlador
);

-- B) Tabla de Tickets
CREATE TABLE tickets (
    id SERIAL PRIMARY KEY,
    titulo VARCHAR(200) NOT NULL,
    descripcion TEXT,
    ubicacion VARCHAR(150),
    categoria VARCHAR(100),
    estatus VARCHAR(50) DEFAULT 'abierto',
    prioridad VARCHAR(50) DEFAULT 'media',
    votos INTEGER DEFAULT 0,
    comentarios TEXT,
    fecha_creacion TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    fecha_actualizacion TIMESTAMP,
    fecha_cierre TIMESTAMP,
    usuario_id INTEGER REFERENCES usuarios(id) ON DELETE SET NULL
);

-- C) Tabla de Historial de Votos
CREATE TABLE votos_registro (
    id SERIAL PRIMARY KEY,
    ticket_id INTEGER REFERENCES tickets(id) ON DELETE CASCADE,
    usuario_id INTEGER REFERENCES usuarios(id) ON DELETE CASCADE,
    UNIQUE(ticket_id, usuario_id)
);

===================================================
 NOTAS TÉCNICAS Y DESPLIEGUE (ACTUALIZADO 2026)
===================================================

1. SEGURIDAD:
   - Las nuevas contraseñas creadas desde la App se guardan encriptadas con bcryptjs.
   - El sistema soporta contraseñas antiguas en texto plano para usuarios legacy.

2. CONEXIÓN EXTERNA (Túnel):
   - Para acceso desde cualquier red/internet: Ejecutar 'ngrok http 3000'.
   - El link devuelto (https://...) debe pegarse en frontend/src/config.js.

3. MODO PRODUCCIÓN:
   - Al ejecutar 'npm run build', el backend sirve automáticamente el frontend.
   - Ya no es necesario usar el puerto 5173 en producción; todo corre por el 3000.
